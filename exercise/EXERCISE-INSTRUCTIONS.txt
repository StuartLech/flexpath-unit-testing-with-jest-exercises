Exercise 9:

Answer:
toBe checks strict identity, so two object variables only pass if they point to the exact same object reference. toEqual checks deep equality by comparing object values. toStrictEqual is stricter than toEqual because it also checks things like undefined properties and object types. See tests/exercise9.test.js for working examples.

Exercise 15:

Answer:
The threshold configuration is documented in jest.config.js. It sets minimum coverage targets of 90% for statements, functions, and lines, plus 80% for branches. When enabled and run with coverage, Jest fails if coverage drops below those thresholds.

Exercise 16:

Answer:
Statement coverage shows whether executable statements ran. Branch coverage checks if alternate paths such as if/else branches ran. Function coverage confirms functions were called. Line coverage reports executed source lines. Together, they reveal untested areas and help assess confidence in bookManager.js.

Exercise 17:

Answer:
Coverage thresholds define the minimum acceptable test coverage. They enforce quality by causing the test run to fail when coverage falls below the target, which helps prevent poorly tested changes from being accepted.
